# Русский сканер рынка с разбором от ИИ

Готовое полнофункциональное приложение для **постоянного анализа крипторынка** с двумя слоями:

1. **Стратегический движок** считает рынок по правилам тренда и риск-менеджмента.
2. **Слой ИИ** поверх сигнала делает понятный разбор: подтверждает идею, выделяет риски, описывает стиль входа и выхода.

> В проекте **нет автоторговли** и **нет подключения к реальному биржевому аккаунту**. Это приложение для аналитики, сигналов и дисциплинированного торгового плана.

## Что умеет приложение
- регулярно загружает свечи по выбранным инструментам;
- считает EMA, RSI, ATR, ADX, моментум, волатильность и относительный объём;
- фильтрует боковик и перегретые входы;
- формирует сигналы `BUY / SELL / HOLD` только после риск-фильтров;
- строит торговый план: вход, стоп, цель 1, цель 2, риск и ориентировочный размер позиции;
- сохраняет историю сигналов;
- показывает статус анализатора и последние результаты во фронтенде;
- при наличии `OPENAI_API_KEY` автоматически добавляет к новым сигналам разбор от ИИ;
- подходит для деплоя на Railway через GitHub.
- в корне уже лежит `railway.json`, поэтому Railway автоматически подхватит команды сборки, запуска и healthcheck.

## Стек
- Фронтенд: React + Vite + TypeScript
- Бэкенд: Node.js + Express + TypeScript
- Источник рыночных данных: публичный Bybit Market Kline API
- ИИ: OpenAI Responses API через серверный SDK
- Хранение: JSON-файл на диске, который позже можно заменить на PostgreSQL

## Развёртывание на Railway

### 1. Переменные окружения
Создайте переменные в Railway:

```env
PORT=3001
MARKET_SYMBOLS=BTCUSDT,ETHUSDT,SOLUSDT
MARKET_TIMEFRAMES=15,60
SCAN_INTERVAL_MS=60000
BYBIT_CATEGORY=linear
HISTORY_LIMIT=500
CORS_ORIGIN=*
ACCOUNT_SIZE_USD=10000
RISK_PER_TRADE_PCT=1
MIN_CONFIDENCE_ACTIONABLE=0.67
OPENAI_API_KEY=ваш_ключ
OPENAI_MODEL=gpt-5.4-mini
AI_ANALYSIS_ENABLED=true
AI_ANALYZE_HOLD_SIGNALS=false
```

### 2. Что уже добавлено в проект

В корне проекта уже есть `railway.json` со следующими настройками:
- `buildCommand`: `npm install && npm run build`
- `startCommand`: `npm start`
- `healthcheckPath`: `/health`
- `numReplicas`: `1`, чтобы не запускать несколько анализаторов одновременно

Это значит, что после импорта репозитория в Railway обычно не нужно вручную прописывать команды в Settings. Railway поддерживает config-as-code через `railway.json` или `railway.toml`, а также отдельные поля для `buildCommand`, `startCommand` и `healthcheckPath`.

### 3. Если хотите указать команды вручную


## Как работает анализ рынка

На каждом цикле сканирования бэкенд:
1. получает исторические свечи по каждому символу и таймфрейму;
2. считает EMA(20), EMA(50), EMA(200), RSI(14), ATR(14), ADX(14), изменение цены, волатильность и относительный объём;
3. определяет режим рынка: восходящий тренд, нисходящий тренд или диапазон;
4. усиливает сигнал только при подтверждении от ADX, импульса и объёма;
5. понижает уверенность в боковике, при перегреве RSI и при слишком высокой волатильности;
6. при качественном сетапе строит торговый план с фиксированным риском на сделку;
7. если подключён OpenAI, отправляет компактный снимок рынка в слой ИИ и получает структурированный разбор по JSON-схеме.

## Что делает слой ИИ

ИИ не заменяет стратегию и не «угадывает рынок с нуля».
Он получает уже готовый снимок рынка:
- текущий сигнал `BUY / SELL / HOLD`;
- уверенность и итоговую оценку;
- режим рынка;
- индикаторы;
- причины сигнала, рассчитанные стратегическим движком;
- торговый план.

После этого OpenAI возвращает:
- свой итоговый вывод;
- оценку уверенности;
- краткое резюме;
- описание рыночного сценария;
- список сильных сторон сетапа;
- список рисков;
- проверочный список перед входом;
- стиль входа;
- стиль выхода;
- условия отмены идеи;
- заметку по размеру позиции.

## Правила стратегии, зашитые в приложение

### 1. Фильтр структуры
- `BUY` допускается, когда EMA20 > EMA50 > EMA200 и цена выше EMA200.
- `SELL` допускается зеркально в нисходящем тренде.
- Если структура смешанная, система предпочитает `HOLD`.

### 2. Фильтр боковика
- ADX должен быть выше рабочего порога.
- При низком ADX приложение не продвигает идею входа как качественный сетап.

### 3. Подтверждение импульса и объёма
- Положительный моментум и объём выше среднего усиливают `BUY`.
- Отрицательный моментум и повышенный объём усиливают `SELL`.

### 4. Защита от запоздалых входов
- Если RSI слишком высок для лонга или слишком низок для шорта, уверенность снижается.
- Если цена слишком далеко ушла от EMA20 относительно ATR, сценарий считается перегретым.

### 5. Риск-менеджмент
- Риск на сделку задаётся в `%` от условного депозита.
- Стоп строится на основе ATR и ближайшей рыночной структуры.
- Цели рассчитываются как `1.5R` и `2.5R`.
- Если сигнал не проходит риск-фильтры, торговый план не выдаётся.

## API и маршруты

### `GET /api/health`
Статус сервера, анализатора и конфигурации слоя ИИ.

### `GET /api/overview`
Сводка по последним сигналам, настройкам риска и статусу слоя ИИ.

### `GET /api/signals`
История сигналов.

Параметры:
- `symbol`
- `timeframe`
- `limit`

### `GET /api/signals/latest`
Последний сигнал по каждому символу и таймфрейму.

### `GET /api/strategy`
Правила стратегии и параметры риск-менеджмента для фронтенда.

### `GET /api/ai/status`
Готовность слоя OpenAI.

## Важные замечания
- Если `OPENAI_API_KEY` не задан, приложение всё равно работает — просто без разборов от ИИ.
- По умолчанию ИИ не анализирует сигналы ожидания, чтобы не тратить токены на слабый рынок.
- Все запросы к OpenAI выполняются **только на сервере**. Ключ никогда не должен попадать во фронтенд.
- Сигналы приложения — это аналитический слой. Рыночные гэпы, проскальзывание и ликвидность могут сделать реальное исполнение хуже рассчитанного сценария.
