
# Русский сканер фьючерсов: что купить, как проверить сигналы и как прогнать paper trading

Это приложение следит за ликвидными USDT-фьючерсами, ищет long-сетапы и показывает простым языком:
- что покупать сейчас;
- что пока только ждать;
- где вход, стоп и две цели;
- что происходит с виртуальными сделками;
- как выглядит бэктест по тем же правилам.

## Что нового в этой версии

Добавлено две проверочные системы:

### 1. Paper trading
Приложение само ведёт виртуальные сделки по живым сигналам:
- открывает виртуальный long, когда появляется `Покупать сейчас`;
- закрывает сделку по стопу, цели 2 или по сигналу `выходить`;
- считает баланс, винрейт, прибыль и комиссии.

### 2. Бэктест
Можно нажать запуск бэктеста и получить проверку стратегии на исторических свечах:
- берутся ликвидные монеты из текущего фьючерсного рынка;
- для каждой монеты стратегия прогоняется по истории;
- считается число сделок, win rate, итог по депозиту, profit factor и max drawdown.

## Важное ограничение

Это **не гарантия прибыли** и не автоторговля. Бэктест и paper trading нужны, чтобы **проверить**, имеет ли стратегия хоть какой-то смысл, прежде чем думать о реальных деньгах.

## API

- `GET /health` — состояние сервиса
- `GET /api/overview` — общая сводка
- `GET /api/opportunities` — лучшие идеи сейчас
- `GET /api/paper` — состояние виртуальных сделок
- `POST /api/paper/reset` — сбросить paper trading
- `GET /api/backtest` — последний результат бэктеста
- `POST /api/backtest/run` — запустить новый бэктест

### Пример запуска бэктеста

```bash
curl -X POST https://ваш-домен.railway.app/api/backtest/run   -H "Content-Type: application/json"   -d '{"maxSymbols":6,"candles":320,"timeframes":["15","60"]}'
```

## Railway

Проект рассчитан на обычную сборку исходников перед запуском:

Если используете `railway.json`, оставьте его в репозитории.


## Выгрузка полной статистики

В интерфейсе есть одна кнопка **«Выгрузить полную статистику»**. Она скачивает один JSON-файл, в котором лежат:
- все сохранённые сигналы,
- текущее состояние демо-счёта,
- открытые и закрытые виртуальные сделки,
- сводная статистика,
- последнее состояние бэктеста.

Этот файл можно загрузить в чат для разбора ошибок и улучшения стратегии.

## Установка на телефон и push-уведомления

В этой версии приложение стало PWA:
- есть `manifest.webmanifest`;
- есть `service worker`;
- можно добавить приложение на главный экран телефона;
- можно включить push-уведомления о новых сигналах `Покупать`.

### Как включить на телефоне

1. Откройте сайт приложения на телефоне.
2. Установите приложение:
   - Android/Chrome: нажмите кнопку установки или пункт меню **Установить приложение**.
   - iPhone/Safari: нажмите **Поделиться** → **На экран “Домой”**, затем откройте приложение уже с иконки.
3. Внутри приложения нажмите **Включить push**.
4. Разрешите уведомления.
5. Нажмите **Тест push**, чтобы проверить доставку.

### Когда приходят push

Push приходит только по новому сигналу `Покупать`. В уведомлении есть:
- монета;
- зона входа;
- стоп;
- TP1;
- TP2.

Чтобы не спамить одним и тем же сигналом, повторное уведомление по той же монете и таймфрейму ограничено параметром:

```env
PUSH_MIN_REPEAT_MS=21600000
```

По умолчанию это 6 часов.

### VAPID-ключи

Для Railway лучше один раз сгенерировать ключи и сохранить их в переменных окружения:

```bash
npm run generate:vapid
```

Затем добавить в Railway Variables:

```env
PUSH_ENABLED=true
PUSH_SUBJECT=mailto:ваш-email@example.com
VAPID_PUBLIC_KEY=...
VAPID_PRIVATE_KEY=...
```

Если ключи не заданы, сервер сам создаст `vapid-keys.json` рядом с файлом storage. Для стабильной работы на Railway лучше использовать постоянные env-ключи.
